## Descricao

Configurar o conector Salesforce no Airbyte para ingerir os objetos CRM padrao: Lead, Opportunity, Contact, Account, Activity (Task + Event). Usar Salesforce Bulk API 2.0 para carga inicial do historico completo (potencialmente milhoes de registros em producao) e streaming incremental para atualizacoes. Mapear os objetos para o modelo Customer 360 com foco nas dimensoes de jornada e financeiro.

Salesforce e o CRM enterprise por excelencia: clientes de maior porte que prospectarem o Elexia Cortex provavelmente ja usam Salesforce. A capacidade de integrar e analisar dados do Salesforce ao lado de outras fontes e um argumento de venda poderoso para o segmento enterprise.

## Criterios de Aceitacao

- [ ] AC1: Conector Salesforce configurado no Airbyte com autenticacao OAuth 2.0 (Connected App); permissoes: `api`, `refresh_token`, `offline_access`
- [ ] AC2: Streams habilitados: `Lead`, `Opportunity`, `Contact`, `Account`, `Task`, `Event`
- [ ] AC3: Carga inicial usando Salesforce Bulk API 2.0 para todos os streams; dados em `bronze.sf_lead`, `bronze.sf_opportunity`, `bronze.sf_contact`, `bronze.sf_account`, `bronze.sf_task`, `bronze.sf_event`
- [ ] AC4: Sincronizacao incremental usando `SystemModstamp` como cursor; processando apenas registros modificados desde a ultima sync
- [ ] AC5: Mapeamento Account → Company e Contact → Customer para identity resolution: `Contact.Email` e `Account.Website` usados como sinais de identidade
- [ ] AC6: Mapeamento de oportunidades para dimensao financeiro do Customer 360: `Opportunity.Amount` → pipeline_value; `Opportunity.StageName = 'Closed Won'` → receita fechada
- [ ] AC7: Mapeamento de atividades para dimensao jornada: Tasks e Events enriquecem `touchpoints` do Customer 360 com historico de interacoes de vendas

## Riscos

- **Bulk API 2.0 com timeout em datasets grandes:** jobs de query longa podem expirar; mitigação: implementar polling com timeout de 30 min e retry automático para jobs que expirem.
- **Custom Objects por cliente:** embora fora do escopo, clientes que dependem de Custom Objects para dados críticos podem se frustrar com o escopo limitado; mitigação: documentar claramente o escopo na demo e roadmap de customização.
- **Salesforce Developer Edition com limites de API:** 15.000 chamadas de API por dia; carga inicial pode consumir cota; mitigação: usar Bulk API 2.0 que não conta nas chamadas de API REST regulares.
- **Lead vs Contact: duplicidade no Customer 360:** um mesmo prospect pode existir como Lead E Contact; mitigação: implementar deduplicação por email na camada Silver antes de inserir no Customer 360.

## Notas Tecnicas

- Salesforce Connected App: criar em `Setup > App Manager > New Connected App`; scopes: `api refresh_token offline_access`; callback URL: URL do Airbyte
- Bulk API 2.0: usar para queries com mais de 10.000 registros; endpoint `POST /services/data/v57.0/jobs/query`; polling ate status `JobComplete`
- Incremental cursor: `SystemModstamp` e mais confiavel que `LastModifiedDate` porque inclui alteracoes de sistema (triggers, workflows)
- Relacionamento Contact → Account: usar `AccountId` para enriquecer Customer 360 com dados da empresa (Account)
- Lead vs Contact: Lead e pre-conversao (prospect); Contact e pos-conversao (cliente); ambos devem ser incluidos no Customer 360 com status diferente
- Salesforce Developer Edition: https://developer.salesforce.com/signup (gratuito, sem expiracao)
- Referencia: https://docs.airbyte.com/integrations/sources/salesforce
