## Descricao

Implementar o conector Omie para ingestao de dados do ERP mais popular entre PMEs brasileiras. O Omie nao possui conector nativo no Airbyte, portanto sera necessario desenvolver um custom connector usando o Airbyte CDK. Os dados a ingerir sao: Clientes, Pedidos de Venda, Contas a Receber e Produtos. O conector deve usar autenticacao via App Key e App Secret da API Omie.

Omie e estrategico para o mercado brasileiro: dezenas de milhares de PMEs usam Omie como ERP principal, e a maioria nao tem nenhuma ferramenta de analytics. O Cortex com conector Omie e a proposta "plug-and-play" para esse segmento.

## Criterios de Aceitacao

- [ ] AC1: Custom connector Omie desenvolvido usando Airbyte CDK (Python); disponibilizado como imagem Docker no ECR do projeto
- [ ] AC2: Autenticacao via App Key + App Secret configurada; credenciais armazenadas no Vault; conector valida credenciais ao inicializar
- [ ] AC3: Streams implementados: `clientes` (cadastro completo), `pedidos` (cabecalho + itens), `contas_a_receber` (titulos, vencimentos, status de pagamento), `produtos` (catalogo)
- [ ] AC4: Sincronizacao incremental implementada usando `dDtAlteracao` (campo de data de alteracao da API Omie) como cursor; full refresh disponivel como fallback
- [ ] AC5: Rate limiting respeitado: API Omie limita a 4 requisicoes/segundo por App Key; conector implementa backoff exponencial com retry em caso de 429
- [ ] AC6: Dados em `bronze.omie_clientes`, `bronze.omie_pedidos`, `bronze.omie_contas_receber`, `bronze.omie_produtos`
- [ ] AC7: Mapeamento para Customer 360 BR: CPF/CNPJ → identidade; contas a receber → inadimplencia e LTV; pedidos → historico de compras; endereco → dimensao identidade
- [ ] AC8: Documentacao de field mapping em `docs/integrations/omie-field-mapping.md` incluindo tratamento de CNPJ vs CPF (pessoa fisica vs juridica)

## Notas Tecnicas

- Omie API: REST, base URL `https://app.omie.com.br/api/v1/`; autenticacao por campo `app_key` e `app_secret` no body de cada request (nao no header)
- Airbyte CDK: usar `airbyte-cdk` versao 0.80+; implementar `FullRefreshStream` e `IncrementalStream`; documentacao em https://docs.airbyte.com/connector-development/cdk-python/
- Paginacao Omie: parametros `pagina` e `registros_por_pagina` (max 50 por chamada); iterar ate `total_de_paginas`
- Tratamento CPF vs CNPJ: campo `pessoa_fisica` (S/N) determina tipo; CPF mascara `###.###.###-##`; CNPJ mascara `##.###.###/####-##`; normalizar removendo mascara antes de tokenizar
- Campos de inadimplencia: `contas_receber.status_titulo` pode ser `RECEBIDO`, `VENCIDO`, `A_VENCER`; `VENCIDO` indica inadimplencia
- Imagem Docker: `{ecr-account}.dkr.ecr.sa-east-1.amazonaws.com/airbyte-source-omie:{version}`
