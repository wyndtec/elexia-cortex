## Descricao

Instalar e configurar o Cube.dev como semantic layer do Elexia Cortex, conectado ao ClickHouse como fonte de dados principal. Definir os cubes (entidades de negocio) e measures (metricas) pre-construidas para as quatro verticais: CRM/Vendas, Varejo/E-commerce, SaaS B2B e Financeiro. Configurar pre-aggregations para garantir performance de consulta abaixo de 2 segundos para o ambiente de demo. Expor APIs REST e GraphQL auto-geradas para consumo pelo frontend.

O Cube.dev e o que permite que o Cortex Intelligence (AI Chat) responda perguntas como "qual e o LTV dos meus top 10 clientes?" sem precisar escrever SQL â€” o Cube traduz a semantica de negocio para queries otimizadas automaticamente. Sem o semantic layer, o chat seria impossivel de implementar de forma confiavel.

## Notas Tecnicas

- Cube.dev: versao 0.35+; schema em TypeScript (nao YAML) para type safety
- Conexao ClickHouse: usar `@cubejs-backend/clickhouse-driver`; `database: 'gold'`; conexao via SSL
- JWT: secret compartilhado entre Cube e Hono.js (L4 API); payload inclui `tenantId` para row-level filtering no Cube
- Tenant isolation no Cube: usar `queryRewrite` para injetar `WHERE tenantId = ${securityContext.tenantId}` automaticamente em todas as queries
- Pre-aggregations: `originalSql` type para views de alto volume; refresh every `1 hour`; granularities: day, week, month
- SaaS metrics: MRR = `SUM(revenue) WHERE billing_period = 'monthly'`; Churn Rate = `churned_customers / beginning_customers * 100`
- Referencia: https://cube.dev/docs/schema/reference
