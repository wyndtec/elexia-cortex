# Story 0.13 — L4 Dashboards Pre-Construidos e Cortex Intelligence (AI Chat Nivel 1)

**Epic:** 0 — Fase 0: Ambiente e Demo
**Status:** Draft
**Prioridade:** P1
**Estimativa:** XL
**Dependencias:** 0.12 (Cube.dev semantic engine), 0.11 (AI Engine — para scores de churn/health), 0.14 (dados sinteticos — para popular os dashboards)

---

## Descricao

Construir a camada de experiencia do Elexia Cortex: quatro dashboards pre-construidos por vertical (CRM/Vendas, Varejo/E-commerce, SaaS B2B, Financeiro) usando React/TypeScript consumindo o Cube.dev semantic layer. Implementar o Cortex Intelligence como AI Chat de Nivel 1: interface de linguagem natural que traduz perguntas do usuario em queries Cube e retorna respostas com visualizacoes. Adicionar alertas configuraveis por threshold e export controlado por RBAC.

Esta story e o produto visivel — tudo que o cliente ve na demo. O objetivo e que em 30 minutos de demo, o prospect veja seu proprio Customer 360, receba predicoes de churn e consiga fazer perguntas em portugues sobre seus dados. O "uau" da demo acontece aqui.

## Criterios de Aceitacao

- [ ] AC1: Dashboard CRM/Vendas: widgets — pipeline por estagio (funil), taxa de conversao por periodo, top 10 clientes por LTV, atividades recentes, churn risk alerts (clientes com score > 70)
- [ ] AC2: Dashboard Varejo/E-commerce: widgets — GMV por periodo, taxa de recompra, ticket medio, abandono de carrinho rate, top produtos por receita, mapa de calor de vendas por regiao (se CEP disponivel)
- [ ] AC3: Dashboard SaaS B2B: widgets — MRR/ARR e tendencia, churn rate e cohort retention, health score distribution, NRR, top clientes em risco, expansao vs contracao
- [ ] AC4: Dashboard Financeiro: widgets — total a receber, taxa de inadimplencia, aging de contas (0-30, 31-60, 61-90, 90+ dias), LTV medio, projecao de receita 30/60/90 dias
- [ ] AC5: Todos os dashboards sao mobile-responsive (320px a 1440px); usam sistema de grid de 12 colunas; dark mode e light mode suportados
- [ ] AC6: Alertas configuraveis: usuario define threshold (ex: "alertar quando churn score > 80"), canal de notificacao (email ou in-app); alertas visiveis no dashboard com badge
- [ ] AC7: Export controlado por RBAC: Analyst e acima podem exportar CSV/PDF; Viewer so pode ver; exports tem watermark com nome do usuario e data para auditoria
- [ ] AC8: Cortex Intelligence (AI Chat Nivel 1): interface de chat no dashboard; usuario digita pergunta em PT-BR; sistema traduz para query Cube via LLM (GPT-4o ou Claude); retorna resposta em texto + visualizacao quando aplicavel
- [ ] AC9: Cortex Intelligence suporta ao menos 20 perguntas predefinidas testadas (exemplos: "quais sao meus clientes com maior risco de churn?", "qual foi meu GMV no ultimo mes?", "mostre os 10 clientes com maior LTV"); respostas corretas em todas
- [ ] AC10: Tempo de carregamento inicial do dashboard < 3 segundos; queries do Cube < 2 segundos (via pre-aggregations)

## Notas Tecnicas

- Frontend: React 18 + TypeScript; bundler Vite; componentes de chart: Recharts ou Tremor (prefereir Tremor por ser opinionated e mais rapido de implementar)
- Design: sistema de design baseado em Tailwind CSS; zero icones em caixas coloridas; monocromatico com acentos na cor primaria da marca
- Cube.dev queries: usar `@cubejs-client/react` hooks (`useCubeQuery`) para queries declarativas no componente
- Cortex Intelligence: LLM escolhido recebe (1) schema do Cube em contexto, (2) pergunta do usuario, (3) historico recente; retorna JSON com `{cube: '...', measures: [...], dimensions: [...], filters: [...]}`; frontend executa a query e renderiza resultado
- LLM para o chat: usar OpenAI GPT-4o via API key armazenada no Vault; fallback para Claude 3.5 Sonnet se rate limit atingido
- Export PDF: usar `react-pdf` ou `@react-pdf/renderer`; watermark como texto em diagonal com opacidade 0.15
- Alertas: armazenar configuracoes em tabela `gold.alert_configs`; job Python avalia thresholds a cada hora; notificacao in-app via WebSocket ou polling
