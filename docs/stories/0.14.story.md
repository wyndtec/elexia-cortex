## Descricao

Criar o ambiente de demo completo do Elexia Cortex: datasets sinteticos realistas para cada vertical (CRM/Vendas, Varejo/E-commerce, SaaS B2B, Financeiro), scripts de seed para popular o banco, um wizard de onboarding guiado para novos usuarios e um "demo mode" que ativa dados ficticiios de qualidade para apresentacoes a prospects.

O ambiente de demo e o que permite vender antes de ter clientes reais conectados. Um dataset sintetico de qualidade — com padroes reais de churn, sazonalidade, cohorts, inadimplencia — deve ser indistinguivel de dados reais para um prospect que ve pela primeira vez. A qualidade do dado sintetico e diretamente proporcional a qualidade da demo.

## Criterios de Aceitacao

- [ ] AC1: Dataset sintetico CRM/Vendas: 1.000 contacts, 500 deals (funil completo), 5 stages, 12 meses de historico; padroes de churn reais (20% churn anual); sazonalidade simulada (Q4 maior volume)
- [ ] AC2: Dataset sintetico Varejo/E-commerce: 2.000 clientes, 8.000 pedidos, 200 produtos (20 categorias), 18 meses de historico; padroes de recompra (30% retention), abandono de carrinho (65%), sazonalidade (Black Friday, Natal)
- [ ] AC3: Dataset sintetico SaaS B2B: 300 contas, MRR crescendo de R$50k a R$200k em 24 meses, churn mensal 2%, expansao de 5%; cohorts visiveis com diferentes taxas de retencao
- [ ] AC4: Dataset sintetico Financeiro (Omie/QBO): 500 clientes, 3.000 notas fiscais/invoices, 12 meses; inadimplencia de 8%, aging realista, ticket medio e distribuicao de pagamentos normais
- [ ] AC5: Scripts de seed em Python: `scripts/seed-demo.py --vertical [crm|ecommerce|saas|financeiro|all] --tenant [tenant_id]`; executa em menos de 2 minutos para todos os verticais
- [ ] AC6: Todos os dados sinteticos passam pelo pipeline dbt (Bronze → Silver → Gold) e produzem um Customer 360 populado; AI Engine gera predicoes para todos os clientes sinteticos
- [ ] AC7: Wizard de onboarding: fluxo de 5 passos para novo usuario — (1) escolha de vertical, (2) escolha de conector ou demo mode, (3) preview de dados, (4) customizacao de KPIs prioritarios, (5) dashboard pronto; completo em menos de 5 minutos
- [ ] AC8: Demo mode: toggle no dashboard que ativa dados sinteticos para apresentacao; dados ficticios usam nomes de empresas brasileiras ficticias (nao reais); CNPJ/CPF invalidos propositalmente para nao coincidir com entidades reais
- [ ] AC9: Qualidade dos dados sinteticos: distribuicoes estatisticas plausíveis (Pareto 80/20 em receita por cliente, distribuicao normal em ticket medio, Poisson em frequencia de compra); revisado manualmente por alguem com experiencia em cada vertical

## Riscos

- **Dados sintéticos com padrões irrealistas:** distribuições mal calibradas podem fazer o demo parecer "artificial" para prospects experientes com dados reais do setor; mitigação: revisar os datasets com alguém que tenha operado em cada vertical antes da primeira demo.
- **Scripts de seed demorando mais de 2 min:** geração de 8.000 pedidos + relações pode ser lenta com Faker; mitigação: pré-gerar os DataFrames e serializar em Parquet para carga rápida; seed carrega Parquet em vez de gerar on-demand.
- **CNPJ/CPF fictício coincidindo com real:** geração com dígito verificador inválido propositalmente é a mitigação, mas precisa de teste explícito; mitigação: rodar validador de CPF/CNPJ real contra todos os registros gerados antes de usar em demo.
- **Wizard de onboarding com estado apenas local:** refresh da página perde o progresso do wizard; mitigação: salvar estado do wizard em localStorage + indicador visual de progresso salvo.

## Notas Tecnicas

- Geracao sintetica: usar biblioteca Python `Faker` (localidade `pt_BR`) para nomes, emails, telefones, enderecos; `numpy.random` para distribuicoes estatisticas; `pandas` para construir DataFrames
- Padroes de churn: usar distribuicao Weibull para modelar tempo de vida de cliente (shape=1.5, scale=365 dias); clientes com `days_since_last_order > 90 AND orders_count < 3` sao marcados como churned
- Sazonalidade: multiplicador mensal aplicado ao volume de pedidos: Jan=0.7, Fev=0.8, Mar=0.9, Abr=0.9, Mai=0.9, Jun=0.95, Jul=0.85, Ago=0.9, Set=0.9, Out=0.95, Nov=1.8 (Black Friday), Dez=1.5 (Natal)
- CNPJs ficticios: gerar com digito verificador invalido propositalmente para nao conflitar com CPF/CNPJ reais
- Wizard de onboarding: implementar como fluxo multi-step em React com estado local; nao requer backend exceto para salvar preferencia de vertical no perfil do usuario
- Demo mode: variavel de sessao `demoMode: boolean` que instrui o Cube.dev a usar `tenant_id = 'demo'` em todas as queries
