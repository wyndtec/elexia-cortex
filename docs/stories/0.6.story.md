## Descricao

Instalar e configurar o Airbyte como motor central de conectores no EKS, e configurar o conector HubSpot como primeiro conector real da Wave 1A. O Airbyte sera o backbone de ingestao de dados de todas as fontes externas. O conector HubSpot deve sincronizar os objetos CRM principais (Contact, Deal, Company, Activity) com suporte a sincronizacao incremental via CDC (Change Data Capture).

HubSpot e o conector prioritario porque a maioria dos prospects do Elexia Cortex usa HubSpot como CRM primario, tornando-o o caso de uso mais impactante para a demo de vendas.

## Notas Tecnicas

- Airbyte versao: 0.50+ (ultima estavel com suporte a ClickHouse destination)
- HubSpot Private App: necessita escopo `crm.objects.contacts.read`, `crm.objects.deals.read`, `crm.objects.companies.read`, `crm.objects.engagements.read`
- ClickHouse destination: usar JDBC driver; `database=bronze`; `ssl=true`
- Airbyte resources no EKS: requests `cpu: 500m, memory: 1Gi`; limits `cpu: 2, memory: 4Gi` por worker
- Temporal (workflow engine do Airbyte): requer PostgreSQL; provisionar RDS PostgreSQL `db.t3.micro` ou usar ClickHouse como alternativa para metadados
- Field mapping HubSpot â†’ Bronze: `hs_object_id` â†’ `source_id`, `email` â†’ tokenizar antes de persistir, `createdate` â†’ `created_at`, `lastmodifieddate` â†’ `updated_at`
- Referencia: https://docs.airbyte.com/integrations/sources/hubspot
